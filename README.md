# Student Exam Score Predictor

Predicts a student's final exam score from study hours, attendance, sleep hours, and previous scores. Includes EDA, a baseline, four trained models with an honest comparison, and a Streamlit app for live predictions.

## Usage

1. Inspect and explore the data (optional, already run): `python notebooks/01_data_inspection.py`, `02_eda.py`, `03_preprocessing.py`, `04_baseline.py`
2. Train models (Linear Regression, Random Forest, Decision Tree, XGBoost) and save the best one: `python train.py`
3. Launch the app: `streamlit run app.py`

## Dataset

[StudentPerformanceFactors.csv](https://github.com/mohansaidinesh/Datasets/blob/main/StudentPerformanceFactors.csv) вЂ” 6,607 students, 20 columns. We use 4 features: `Hours_Studied`, `Attendance`, `Sleep_Hours`, `Previous_Scores`, predicting `Exam_Score`.

## Results

| Model | MAE | RMSE | RВІ |
|---|---|---|---|
| Baseline (mean) | 2.82 | 3.76 | -0.001 |
| **Linear Regression** | **1.36** | **2.31** | **0.622** |
| Decision Tree | 1.60 | 2.51 | 0.554 |
| Random Forest | 1.57 | 2.61 | 0.516 |
| XGBoost | 1.81 | 3.04 | 0.348 |

Linear Regression won вЂ” the feature-target relationships are close to linear/monotonic, so the added flexibility of tree-based models (Decision Tree, Random Forest, XGBoost) didn't pay off, and the more complex models (especially XGBoost) overfit on just 4 features.

## Limitations

- Only 4 of 20 available features used; richer feature sets (tutoring, motivation, parental involvement) could improve results.
- Sleep_Hours showed no real relationship with the target in this dataset вЂ” don't read too much into its coefficient.
- Single train/test split (no cross-validation); results may vary slightly with different splits.

## Future work

- Try cross-validation and hyperparameter tuning (e.g., GridSearchCV on Random Forest/XGBoost).
- Include categorical features (Parental_Involvement, Motivation_Level, etc.) via one-hot encoding.
